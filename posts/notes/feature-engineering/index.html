Preprocessing

  1. Data Cleansing: To eliminate or correct erroneous data by identifying and handling inconsistencies or outliers.

  2. Scaling and Normalizing: Numerical features often need to be scaled or normalized to ensure the models can learn effectively. Scaling adjusts the range of features, while normalization brings them within a specific range, such as between 0 and 1.

  3. Dimensionality Reduction: Involves reducing the number of features by creating a lower-dimensional representation of the data. It helps to remove noise, improve model performance, and interpretability.

  4. Feature Construction: Feature construction involves creating new features from existing ones using various techniques. For example, categorical features can be transformed into one-hot vectors, and numerical features can be mapped to floats. This process helps extract more meaningful information from the data.

Different types of data require specific preprocessing techniques.

For text data, techniques like stemming, lemmatization, TF-IDF, n-grams, and embeddings can be used to enhance predictive quality. Similarly, image data can benefit from operations such as resizing, cropping, filtering, and applying photometric distortions.

Preprocessing plays a crucial role in building accurate predictive models by transforming raw data into informative features.

Feature Engineering Techniques

Feature engineering involves transforming raw input data into a feature vector using various techniques. Here are some commonly used techniques:

  1. Feature Scaling: Scaling numerical features ensures they are within a similar range, making it easier for models to learn from them. Techniques like normalization and standardization are used to scale features effectively.

  2. Bucketizing/Binning: Grouping numerical data into buckets or bins can convert continuous values into discrete categories. This technique is useful when encoding numerical data as categories, enabling models to learn patterns based on these groups.

  3. Dimensionality Reduction: Techniques like Principal Component Analysis (PCA), t-SNE, and UMAP help reduce the number of features while preserving important information. Dimensionality reduction can enhance model performance and simplify data representation.

  4. Feature Crosses: Feature crosses involve combining multiple features to create new features. This technique allows for the encoding of non-linear relationships or the expression of the same information in fewer features. Feature crosses can be formed by multiplying numerical features or combining categorical and numerical features.

  5. Encoding Features: Encoding features involves transforming categorical variables into numerical representations. Techniques like one-hot encoding and categorical embeddings are used to convert categorical data into a format suitable for models.

By applying these feature engineering techniques, raw data can be transformed into more meaningful features, improving model performance and enhancing the understanding of the underlying data patterns.

Key points:
