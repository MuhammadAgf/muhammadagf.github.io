Building a Search Product

January 1, 1001 2-minute read
product • search

So you want to add search engine to your app, before jumping to algorithm and the “how”, just like any product that you want to build, ask “what problem our user is facing?” and “why are our user face that problem?”

Before we start

I assume that you have done your homework and you discover that you need to create a search engine to solve your user problem, but you have many question

  1. what kind of features your search should have?
  2. how many filter and what filter should be implemented in your search?
  3. how many vertical?
  4. how many results should be shown each page?
  5. should your search able to handle typo? semantic similarity?, etc

there are no trivial answer for those question, but before we even jump to that question we should ask:

if the search features is already exist, why our user is make that search? do they look for a specific product they want to buy? or are they casually looking for something? those “why” can be described by a search intent, there are many different types of search intent, generally

Intent

The way each company building a “search engine” can be different, before thinking what kind of features and technology you want to add in your search engine you should first think do you even need a “search engine” or maybe you should create something else?

before jumping to a search products, lets start with user intent
